Sí. Para no andar abriendo Inspeccionar → Network, lo más cómodo es usar una extensión que detecte manifiestos HLS/DASH automáticamente.

Las más útiles:

1. The Stream Detector / Stream Detector Plus
Detecta .m3u8, .mpd, subtítulos y a veces te genera comandos listos para yt-dlp, ffmpeg, streamlink, etc. Es de las mejores para justo lo que dices: copiar la URL sin bucear en DevTools. La versión de Firefox está en Mozilla Add-ons y también existe “Stream Detector Plus” en Chrome Web Store. 

2. Video DownloadHelper
Más “plug and play”. Detecta streams HLS/DASH y permite descargar o convertir en varios formatos, aunque algunas funciones pueden depender de su app complementaria. Soporta DASH, HLS y MPD según su ficha. 

3. HLS Downloader
Sirve más para HLS .m3u8 que para DASH .mpd. Útil cuando la web usa fragmentos .ts, pero menos completo para casos como MPEG-DASH con audio/vídeo separados. 

Para URLs como las que pegaste, ojo: init-v1-x3.mp4 y init-a1-x3.mp4 parecen fragmentos de inicialización DASH, no el vídeo completo. Normalmente necesitas el manifest .mpd o todos los segmentos de audio/vídeo con sus headers/cookies. Ahí “The Stream Detector” suele ahorrar bastante porque intenta capturar el .mpd y generar el comando.

Mi recomendación práctica: Firefox + The Stream Detector para sacar la URL/comando, y luego yt-dlp o ffmpeg si es contenido al que tienes derecho de acceso. Si está protegido con DRM, estas extensiones no deberían saltárselo y lo normal es que no sirva.

Eso ya es el manifest DASH .mpd. Con eso no necesitas las URLs init-* sueltas; el reproductor/descargador puede reconstruir el vídeo usando el SegmentTemplate.

En tu caso hay:

Vídeo: RepresentationID = v1-x3

Audio español: RepresentationID = a1-x3

Duración total: 5641.589 s, unos 94 min 1 s

Segmentos de 10 s, así que habrá aproximadamente 565 fragmentos por pista.

La forma más simple, si tienes derecho a descargarlo, es guardar ese XML como video.mpd y usar:

ffmpeg -i video.mpd -c copy salida.mp4
O con yt-dlp:

yt-dlp -o "salida.%(ext)s" video.mpd
Si el sitio requiere cookies o cabeceras, normalmente sería algo tipo:

yt-dlp --cookies-from-browser firefox -o "salida.%(ext)s" video.mpd
o con un referer:

yt-dlp --referer "https://pagina-original.com/" -o "salida.%(ext)s" video.mpd
También podrías generar una URL de fragmento manualmente sustituyendo:

fragment-$Number$-$RepresentationID$.m4s
por, por ejemplo:

fragment-1-v1-x3.m4s
fragment-1-a1-x3.m4s
fragment-2-v1-x3.m4s
fragment-2-a1-x3.m4s
Pero hacerlo a mano no merece la pena. Mejor darle el .mpd a ffmpeg, yt-dlp o N_m3u8DL-RE.

Importante: las URLs que pusiste antes:

init-v1-x3.mp4
init-a1-x3.mp4
son solo los archivos de inicialización. No contienen todo el vídeo/audio; sirven para que los fragmentos .m4s tengan contexto. El contenido real está en los fragment-N-v1-x3.m4s y fragment-N-a1-x3.m4s.

      color: white;
      font-family: sans-serif;
      font-size: 24px;
      z-index: 10;
    }
  </style>

<body>
  <div id="msg">Cargando...</div>
  <video id="video" controls autoplay></video>

  <script>
    let player;
    let currentUrl = "";

    async function init() {
      shaka.polyfill.installAll();

      const video = document.getElementById("video");
      player = new shaka.Player(video);

      player.addEventListener("error", e => {
        console.error("Error Shaka:", e.detail);
        document.getElementById("msg").textContent = "Error cargando vídeo";
      });

      await checkVideoUrl();

      setInterval(checkVideoUrl, 5000);
    }

    async function checkVideoUrl() {
      try {
        const res = await fetch("/video.txt?cache=" + Date.now());
        const url = (await res.text()).trim();

        if (!url || url === currentUrl) return;

        currentUrl = url;
        document.getElementById("msg").textContent = "Cargando vídeo...";

        if (url.endsWith(".mp4")) {
          const video = document.getElementById("video");
          video.src = url;
          await video.play();
        } else {
          await player.load(url);
        }

        document.getElementById("msg").textContent = "";
      } catch (e) {
        console.error(e);
        document.getElementById("msg").textContent = "No se pudo cargar la URL";
      }
    }

    document.addEventListener("DOMContentLoaded", init);
  </script>
</body>
</html>

Sí, pero no tan directo como con tu HTML + Shaka.

Plex
Para esto, Plex no es buena vía. Plex está pensado para bibliotecas de archivos tipo .mp4, .mkv, etc., y cuando el cliente no soporta algo lo direct-stream/transcodifica desde el servidor. Su documentación habla de compatibilidad por contenedor/códec, no de meterle un .mpd externo como fuente reproducible directa. 

Además, Plex tampoco soporta IPTV/M3U de forma directa sin inventos tipo xTeVe/tuner virtual, y eso normalmente va mejor con HLS .m3u8 que con DASH .mpd. 

Conclusión Plex: no lo usaría para tu caso.

Jellyfin
Con Jellyfin podrías usarlo si primero conviertes/descargas ese DASH a un archivo normal:

video.mp4
video.mkv
y lo metes en tu biblioteca.

Pero como reproductor de un .mpd externo, tampoco es lo ideal. Hay issues de Jellyfin donde se indica que MPD/DASH como fuente Live TV no está soportado o no está planteado directamente. 

Jellyfin sí transcodifica archivos de tu biblioteca con FFmpeg cuando el cliente no soporta un formato, pero eso es distinto a usar un .mpd remoto como “URL pegada y play”. 

Conclusión Jellyfin: mejor que Plex si quieres biblioteca propia, pero no para pegar manifests .mpd dinámicos.

Lo más simple para lo que quieres
Quédate con tu solución:

Android TV → tv.html → manifest.mpd → Shaka Player
Es más simple, más controlable y justo para tu caso.

La mejora sería hacer tu panel admin así:

Pego URL .mpd
o pego XML <MPD>...</MPD>
Guardar
La TV actualiza sola
Plex/Jellyfin solo los usaría si decides convertir cada vídeo a .mp4/.mkv y tener una biblioteca permanente. Para “pegar enlace desde el móvil y verlo en la TV”, tu player web es mejor.

2. guardar.php
Este archivo recibe el MPD, lo guarda y actualiza catalogo.json.

<?php

$PASSWORD = "CAMBIA_ESTA_CLAVE";

$pass = $_POST["pass"] ?? "";

if ($pass !== $PASSWORD) {
    http_response_code(403);
    exit("Clave incorrecta");
}

$title = trim($_POST["title"] ?? "");
$description = trim($_POST["description"] ?? "");
$manifestUrl = trim($_POST["manifest_url"] ?? "");
$manifestXml = trim($_POST["manifest_xml"] ?? "");

if ($title === "") {
    http_response_code(400);
    exit("Falta el título");
}

$catalogFile = __DIR__ . "/catalogo.json";
$manifestDir = __DIR__ . "/manifests";
$posterDir = __DIR__ . "/posters";

if (!is_dir($manifestDir)) {
    mkdir($manifestDir, 0775, true);
}

if (!is_dir($posterDir)) {
    mkdir($posterDir, 0775, true);
}

function slugify($text) {
    $text = strtolower($text);
    $text = iconv("UTF-8", "ASCII//TRANSLIT", $text);
    $text = preg_replace("/[^a-z0-9]+/", "-", $text);
    $text = trim($text, "-");

    if ($text === "") {
        $text = "video";
    }

    return $text;
}

$slug = slugify($title);
$unique = $slug . "-" . time();

$manifestPathPublic = "";
$posterPathPublic = "";

/*
  Caso 1: pegar URL .mpd
*/
if ($manifestUrl !== "") {
    if (!filter_var($manifestUrl, FILTER_VALIDATE_URL)) {
        http_response_code(400);
        exit("La URL del MPD no es válida");
    }

    $manifestPathPublic = $manifestUrl;
}

/*
  Caso 2: subir archivo .mpd
*/
elseif (isset($_FILES["manifest_file"]) && $_FILES["manifest_file"]["error"] === UPLOAD_ERR_OK) {
    $content = file_get_contents($_FILES["manifest_file"]["tmp_name"]);

    if (stripos($content, "<MPD") === false) {
        http_response_code(400);
        exit("El archivo no parece ser un MPD válido");
    }

    $filename = $unique . ".mpd";
    $target = $manifestDir . "/" . $filename;

    file_put_contents($target, $content);

    $manifestPathPublic = "manifests/" . $filename;
}

/*
  Caso 3: pegar XML completo
*/
elseif ($manifestXml !== "") {
    if (stripos($manifestXml, "<MPD") === false) {
        http_response_code(400);
        exit("El texto pegado no parece ser un MPD válido");
    }

    $filename = $unique . ".mpd";
    $target = $manifestDir . "/" . $filename;

    file_put_contents($target, $manifestXml);

    $manifestPathPublic = "manifests/" . $filename;
}

else {
    http_response_code(400);
    exit("Debes pegar una URL, pegar XML o subir un archivo MPD");
}

/*
  Poster opcional
*/
if (isset($_FILES["poster_file"]) && $_FILES["poster_file"]["error"] === UPLOAD_ERR_OK) {
    $tmp = $_FILES["poster_file"]["tmp_name"];
    $info = getimagesize($tmp);

    if ($info === false) {
        http_response_code(400);
        exit("El poster no parece ser una imagen válida");
    }

    $mime = $info["mime"];

    if ($mime === "image/jpeg") {
        $ext = "jpg";
    } elseif ($mime === "image/png") {
        $ext = "png";
    } elseif ($mime === "image/webp") {
        $ext = "webp";
    } else {
        http_response_code(400);
        exit("Formato de poster no soportado");
    }

    $posterName = $unique . "." . $ext;
    $posterTarget = $posterDir . "/" . $posterName;

    move_uploaded_file($tmp, $posterTarget);

    $posterPathPublic = "posters/" . $posterName;
}

/*
  Leer catálogo actual
*/
$catalog = [];

if (file_exists($catalogFile)) {
    $json = file_get_contents($catalogFile);
    $decoded = json_decode($json, true);

    if (is_array($decoded)) {
        $catalog = $decoded;
    }
}

/*
  Añadir nuevo item
*/
$item = [
    "title" => $title,
    "description" => $description,
    "manifest" => $manifestPathPublic,
    "poster" => $posterPathPublic
];

$catalog[] = $item;

/*
  Guardar catálogo
*/
file_put_contents(
    $catalogFile,
    json_encode($catalog, JSON_PRETTY_PRINT | JSON_UNESCAPED_SLASHES | JSON_UNESCAPED_UNICODE)
);

echo "OK";

consume muchos recuros el vps?

No, casi nada, siempre que tu VPS solo sirva:

tv.html
admin.html
guardar.php
catalogo.json
archivos .mpd
posters pequeños
En ese caso el vídeo no pasa por tu VPS. Tu VPS solo entrega la interfaz y el manifest; los fragmentos .m4s los descarga la TV directamente desde el servidor externo del MPD.

Consumo aproximado
Para 1–5 usuarios tuyos:

CPU: casi 0%
RAM: 50–200 MB según contenedor PHP/Apache/Nginx
Disco: muy poco, salvo posters
Ancho de banda: bajísimo
Un MPD suele pesar KB o pocos MB. El tráfico grande, que sería el vídeo, va desde:

servidor externo del vídeo → Android TV
no desde:

VPS → Android TV
Cuándo sí consumiría mucho
Consumiría bastante si haces una de estas cosas:

❌ descargar los vídeos al VPS
❌ guardar todos los fragmentos .m4s
❌ convertir DASH a MP4 en el servidor
❌ hacer proxy de los fragmentos por tu VPS
❌ transcodificar vídeo
Ahí sí podrías gastar CPU, disco y mucho ancho de banda.

Para tu caso
Con Docker + Caddy + PHP, una Oracle Free Tier pequeña va sobrada.

Algo tipo:

1 OCPU
1 GB RAM
10–20 GB disco
te sirve perfectamente para esto.

La única parte que puede pesar un poco son los posters si subes muchas imágenes grandes. Mejor usar .jpg o .webp comprimidos.

Puedes usar un reproductor UI como Plyr encima del <video>. Queda mucho más estético: barra moderna, botón grande, fullscreen, volumen, progreso, etc.

3. Buffer real / configuración de Shaka
También puedes ajustar cuánto buffer intenta guardar Shaka. Por ejemplo:

player.configure({
  streaming: {
    bufferingGoal: 60,
    rebufferingGoal: 5,
    bufferBehind: 30
  }
});
Eso significa algo aproximado a:

bufferingGoal: intenta mantener hasta 60s por delante
rebufferingGoal: espera unos 5s antes de volver a reproducir si se corta
bufferBehind: conserva 30s por detrás
Para Android TV, algo razonable sería:

player.configure({
  streaming: {
    bufferingGoal: 30,
    rebufferingGoal: 4,
    bufferBehind: 20
  }
});
No conviene poner valores enormes porque puede gastar más RAM.
